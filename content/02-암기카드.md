## 🃏 B. 의무대상 / 합격기준 / 응시요건

| 항목 | 핵심 |
|------|------|
| ISMS 의무 (택1) | ① ISP ② IDC ③ 매출 **100억↑** or 이용자 일평균 **100만명↑** 정보통신서비스 ④ 매출 **1,500억↑** 상급종합병원·대규모대학 |
| ISMS-P | **자율**(개인정보 포함) — 의무 아님 |
| 합격(2025) | 절대평가 **60점↑** |
| 합격(2026~) | **상대평가**: **60점↑ 중 상위 100명** 선발 |
| 응시인원 제한 | KISA 당해연도 공고로 확인(공식 미확정 — "2,000명" 설은 미검증) |
| 응시요건 | 4년제졸 + 보호·개인정보 **각 1년** + **합계 6년** |
| 시험구성 | **50문항 / 120분 / 5지선다** |
| 심사원 등급 | 심사원보 < 심사원 < **선임심사원**(팀장) |

## 🃏 C. 암호 알고리즘

| 구분 | 안전 ✅ | 금지 ❌ |
|------|--------|--------|
| 대칭키 | **SEED, ARIA, AES, HIGHT, LEA** | DES, 3DES(폐지진행), RC4 |
| 해시 | **SHA-256/384/512** | SHA-1, MD5, MD4 |
| 공개키 | RSA(2048↑), ECC | RSA-1024↓ |
| (주의) | CRC32 = 무결성용, 암호 아님 | — |

국산 4종 = **세아하리**(SEED·ARIA·HIGHT·LEA).
암호화 저장 대상 = **비밀번호(일방향)·주민번호·고유식별정보·바이오정보·계좌·카드·여권**.

## 🃏 D. 인증기준 영역 구분 (어느 대분류?)

| 분야 | 대분류 | 분야 | 대분류 |
|------|--------|------|--------|
| 경영진 참여(1.1.1) | **1 관리체계** | 직무분리(2.2.2) | **2 보호대책** |
| 위험평가(1.2.3) | **1** | 접근통제(2.6) | **2** |
| 관리체계 점검(1.4.2) | **1** | 암호화(2.7) | **2** |
| 자산식별(1.2.1) | **1** | 재해복구(2.12) | **2** |
| 수집·이용(3.1.1) | **3 개인정보** | 제3자제공(3.3.1) | **3** |
| 파기(3.4.1) | **3** | 권리보장(3.5.2) | **3** |

개수: **관리체계 16 / 보호대책 64 / 개인정보 21 = 101** (ISMS는 80).
흐름: 1번=**PDCA**, 3번=**수집→보유→제공→파기→권리**.

## 🃏 E. 헷갈리는 기준 1:1 비교 (시험 최빈출 함정)

### ① 제3자 제공(3.3.1) vs 위탁(3.3.2) ⭐⭐⭐
| | 제3자 제공 | 위탁 |
|--|-----------|------|
| 누구 목적? | **받는 자**의 이익·목적 | **위탁자(우리)** 목적 |
| 예시 | 제휴 카드사에 정보 넘겨 카드사가 영업 | 택배·콜센터·광고문자 대행 |
| 동의? | **별도 동의 필수** | 동의 불요 |
| 공개? | 동의 시 고지 | **처리방침에 위탁사실·수탁자 공개** |
| 추가의무 | — | 수탁자 **교육·감독·점검**, 문서체결 |
> 판별 한마디: **"누구의 이익을 위해 쓰나"** — 받는 쪽이면 제공, 우리 대신이면 위탁.

### ② 2.5 인증·권한 vs 2.6 접근통제
| 2.5 (누가 들어오나) | 2.6 (어디까지 닿나) |
|--------------------|---------------------|
| 계정·식별·인증·비번·권한부여/검토 | 네트워크/시스템/응용/DB/무선/원격/인터넷 |
| 예: 공용계정 공유→2.5.2 | 예: 개발자 운영DB 접근→2.6.4 |

### ③ 2.5.5 특수권한 vs 2.5.6 권한검토
- **2.5.5** = 관리자 등 특수계정의 **최소부여·식별·관리**
- **2.5.6** = 부여된 권한 **주기적 재검토·회수**

### ④ 2.9.4 로그관리 vs 2.9.5 로그점검
- **2.9.4** = 로그 **생성·보관**(미생성·보관기간 미달)
- **2.9.5** = 로그 **점검·분석**(점검 안 함)

### ⑤ 1.2.3 위험평가 vs 1.4.2 관리체계 점검
- **1.2.3** = 구축단계 위험 식별·평가(연 1회)
- **1.4.2** = 운영단계 내부감사·점검(연 1회)

### ⑥ 점검만 vs 점검+조치
- 취약점 **점검 후 조치 안 함** → **2.11.2**(점검 및 조치)
- 패치 자체 안 함 → **2.10.8**(패치관리)

### ⑦ 2.8.3 환경분리 vs 2.8.4 시험데이터
- **2.8.3** = 개발·운영 환경을 **안 나눔**
- **2.8.4** = 운영 **실데이터를 테스트에 그대로** 사용

### ⑧ 2.2.5 퇴직관리 vs 2.5.1 계정관리 vs 2.5.6 권한검토
- 퇴직 트리거(통보·회수 절차 부재) → **2.2.5**
- 계정 생성·삭제 절차 일반 → **2.5.1**
- 재검토 미수행 → **2.5.6**

### ⑨ 가명정보 vs 익명정보
| | 가명정보 | 익명정보 |
|--|---------|---------|
| 재식별 | 추가정보로 **가능** | **불가능** |
| 법 적용 | **개인정보(적용 O)** | 적용 X |
| 결합 | **전문기관** 통해서만 | 제한 없음 |
| 동의 없이 처리 | 통계·연구·공익기록 목적 | 자유 |

### ⑩ 주민번호(3.1.3/§24의2) vs 고유식별정보(3.1.4/§24) vs 민감정보(3.1.4/§23)
- 주민번호: **법령 근거 없으면 처리 금지**(동의로도 X)
- 고유식별(여권·면허·외국인번호): **별도 동의** 시 가능
- 민감정보(사상·건강·노조 등): **별도 동의** 시 가능

### ⑪ 목적 외 이용(3.2.4) vs 제3자 제공(3.3.1)
- 같은 처리자가 **수집목적과 다른 용도**로 사용 → 3.2.4
- **외부 제3자**에게 넘김 → 3.3.1

### ⑫ 2.10.1 보안시스템 vs 2.11.3 이상행위 모니터링
- IPS/WAF 등 **보안장비 자체 운영·이벤트 검토** → 2.10.1
- 조직 전반 **이상행위 탐지·분석 체계** → 2.11.3

## 🃏 F. 개인정보 수집 6대 적법근거 (§15) ⭐

1. **동의**
2. **법률** 특별규정·법적 의무
3. **계약** 체결·이행 불가피
4. **급박한** 생명·신체·재산 이익
5. **정당한 이익**(정보주체 권리보다 우선 + 합리적 관련성)
6. **공공기관** 소관업무

> 함정: "묵시적 동의(거부 안 함)", "마케팅이니까 당연히", "직원이니까"는 적법근거 **아님**.
